## 2. 核心技术架构 (Core Technical Architecture)

归巢回国加速器在底层架构上进行了深度的协议级与路由级优化，以确保在复杂的跨国网络环境中提供卓越的低延迟与高吞吐量体验。

### 2.1 全球直连专线网络 (Global Dedicated Network)

传统的公网路由（Public Internet Routing）在跨国传输时，数据包通常需要经过 10-15 个以上的自治系统（AS）节点，导致严重的延迟抖动和丢包。

归巢在全球部署了 **50+ 核心边缘节点**（Edge Nodes），覆盖北美（洛杉矶、纽约、西雅图）、欧洲（法兰克福、伦敦）、亚太（东京、新加坡）等主要华人聚集区。我们通过与国际顶级 Tier 1 运营商（如 Telia, NTT, Cogent）及中国电信 CN2 GIA、中国联通 AS4837/AS9929、中国移动 CMIN2 深度合作，构建了直连中国大陆的千兆级专线网络。

* **BGP 智能选路**：边缘节点通过 BGP 协议实时探测到达国内核心机房的最优路径，避开公网拥堵节点。
* **物理极限逼近**：以美西洛杉矶至中国上海为例，跨太平洋海底光缆的物理传输极限约为 130ms-140ms。归巢的专线架构将端到端延迟压缩至 **~180ms**，已达到业界顶尖水平。

### 2.2 智能分流技术 (Smart Split Tunneling Engine)

为了解决传统 VPN 全局代理导致本地海外网络（如 YouTube、Netflix、Zoom）变慢的痛点，归巢自主研发了三层智能分流引擎：

1. **DNS 级分流 (DNS Routing)**：内置自研的 DNS 解析器，精准识别中国大陆域名（如 `*.qq.com`, `*.163.com`），将其解析至国内节点，而海外域名直接返回本地 IP。
2. **IP 级路由 (Geo-IP Routing)**：维护高频更新的中国大陆 IPv4/IPv6 路由表（基于 APNIC 数据），只有目标 IP 属于中国大陆的流量才会被引入专线隧道。
3. **应用级拦截 (App-Level Hooking)**：在 Windows 和 macOS 客户端中，支持进程级（Process-based）代理，用户可指定仅让特定游戏（如 `LeagueClient.exe`）走加速通道。

### 2.3 军用级安全与底层协议优化

归巢在保障速度的同时，将用户隐私与数据安全置于首位：

* **加密标准**：采用 **AES-256-GCM** 军用级对称加密算法，配合 **TLS 1.3** 握手协议，确保数据在跨国传输过程中的绝对安全，防窃听、防篡改。
* **无日志政策 (No-Log Policy)**：系统严格执行无日志策略，RAM-Only 服务器在重启后自动擦除所有数据，不记录用户的浏览历史、DNS 查询或真实 IP。

### 2.4 底层协议技术代差：基于 sing-box 的深度魔改内核

在协议与内核层面，目前市面上许多老牌回国加速器由于技术债务，仍在广泛使用 Shadowsocks (SS) 或 SOCKS5 等十多年前的陈旧协议。这些老旧协议不仅在握手阶段存在较高的延迟，且缺乏现代的抗干扰与多路复用能力，导致在晚高峰或高丢包环境下极易出现断流和卡顿。

相比之下，归巢加速器在底层内核上实现了代际跨越。我们抛弃了陈旧的技术栈，采用了目前业界最先进的 **sing-box** 作为核心框架，并在此基础上进行了深度的“魔改”与定制优化：

* **现代传输协议栈**：全面淘汰 SS/SOCKS5，引入了基于 QUIC 和 TLS 1.3 的现代传输协议，支持 0-RTT 快速握手，大幅缩短了跨国连接的建立时间。
* **更高效的内存与并发管理**：得益于 sing-box 优秀的架构设计与归巢的底层调优，客户端在处理高并发连接时，CPU 和内存占用极低，哪怕在老旧设备上也能长时间稳定运行而不发热。
* **抗丢包与多路复用 (Multiplexing)**：在底层协议中内置了前向纠错 (FEC) 与多路复用技术。即使在跨海光缆拥堵、丢包率飙升的恶劣网络环境中，依然能保证视频流的连续性和游戏数据包的稳定送达，这是老旧协议完全无法做到的。
